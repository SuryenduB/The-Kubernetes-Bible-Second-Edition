L3 VLAN Migration & NAT Gateway Setup

Implementation guide to segment K3s homelab traffic using Netgear GS716Tv3 routing capability and NUC asymmetric NAT.

⚠️ Critical Deployment Safeguards (Verify Before Commencing)

1. Netgear Switch Save config: Netgear switches do not automatically save to NVRAM on application. After any settings adjustments, you must go to Maintenance > Save Config, tick the active slot, and click Apply.

2. K3s Node-IP Bindings: If node IPs change, Kubernetes certificates block communication. Service definitions on the master and workers must have --node-ip explicitly declared before rebooting.

3. Switch Gateway Asymmetry: The default route of the switch points to the NUC's VLAN 1 IP (acting as NAT forwarding engine), which masquerades packets to the Vodafone Router.

1
Phase 1: Pre-Migration Validation

Deploy Link Layer Discovery Protocol to all cluster targets to map physical configurations.

pwsh -File .\WindowsLab\Install-LLDP.ps1

All K3s nodes responded via LLDP. However, speed warnings are registered on g10 (kubernetes7) and g11 (HP-2), which are bottlenecked at 100M. This is a blocking hardware issue that must be resolved by swapping/checking cables before proceeding to Phase 2 of the migration to ensure storage and network health.

Phase 2: Router DHCP Reservations

Since the Vodafone Router cannot perform static routing, it will only serve DHCP leases. Configure the DHCP pools and IP reservations:

Device MAC Address Subnet / VLAN Assigned IP
NUC Master (VLAN 1) C0:3F:D5:6D:45:85 VLAN 1 (Flat) 192.168.0.21
NUC Master (VLAN 20) C0:3F:D5:6D:45:85 VLAN 20 (K3s) 192.168.20.21
HP-1 Registry C8:D3:FF:6A:72:2E VLAN 20 (K3s) 192.168.20.236
HP-2 Workstation FC:3F:DB:86:1A:81 VLAN 30 (Trusted) 192.168.30.122

⚠️ Static Route Verification Not Found

The Vodafone router routing panel is locked down. Proceed with NUC Next-Hop NAT translation architecture (Option A.2).
3
Phase 3: Switch L3 Interface Configuration

Configure Switched Virtual Interfaces (SVIs) and Default Routing on the Netgear GS716Tv3 switch Web Interface:

  1. Navigate to Routing > IP > IP Configuration, set Routing Mode to Enable, and click apply.
  2. Go to Routing > IP > IP Interface Configuration and create routing interface addresses:
    • VLAN 1: 192.168.0.99 (Mask: 255.255.255.0)
    • VLAN 10 (MGMT): 192.168.10.99
    • VLAN 20 (K3s): 192.168.20.99
    • VLAN 30 (Trusted): 192.168.30.99
  3. Set default routing under Routing > IP > Static Route Configuration:
    Destination: 0.0.0.0 Subnet Mask: 0.0.0.0 Next Hop Gateway: 192.168.0.21 (NUC VLAN 1 Interface - NOT Vodafone Router)
  4. Configure DHCP relay under System > Services > DHCP Relay, point helper to 192.168.0.1.

⚠️ Configuration Persistence Requirement

Go to Maintenance > Save Config, check the checkbox and click Apply, or your configuration will vanish upon next powercycle.
4
Phase 4: Port Memberships & Trunk Setup

Assign VLAN IDs and adjust trunking configuration on the switch ports:

  1. Navigate to Switching > VLAN > Basic > VLAN Configuration and create VLAN IDs 10, 20, 30.
  2. Set Port g8 (NUC Link) as trunk:
    • VLAN 1: Set to Untagged (U), PVID = 1.
    • VLAN 20: Set to Tagged (T).
  3. Set K3s nodes ports (g2-g6, g10, g12, g15) to VLAN 20:
    • PVID Configuration: Set PVID to 20.
    • VLAN Membership: Set to Untagged (U) on VLAN 20.
  4. Set HP-2 (g11) to VLAN 30:
    • PVID Configuration: Set PVID to 30.
    • VLAN Membership: Set to Untagged (U) on VLAN 30.
5
Phase 5: NUC VLAN Tagging & IPTables Gateway

Configure Netplan on the NUC to receive tagged VLAN 20 traffic and act as a NAT masquerade engine.

1. Edit /etc/netplan/01-netcfg.yaml (or equivalent netplan config):

network: version: 2 ethernets: enp3s0: dhcp4: true routes: - to: 192.168.10.0/24 via: 192.168.0.99 - to: 192.168.30.0/24 via: 192.168.0.99 vlans: vlan20: id: 20 link: enp3s0 addresses: - 192.168.20.21/24

2. Apply netplan config:

sudo netplan apply

3. Enable IP Forwarding and NAT rules:

# Enable forwarding instantly sudo sysctl -w net.ipv4.ip_forward=1 echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf # Add NAT masquerade rules sudo iptables -t nat -A POSTROUTING -o enp3s0 -j MASQUERADE sudo iptables -A FORWARD -i vlan20 -o enp3s0 -j ACCEPT sudo iptables -A FORWARD -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT # Save iptables rules permanently sudo apt-get install iptables-persistent -y sudo iptables-save | sudo tee /etc/iptables/rules.v4
6
Phase 6: K3s Node Settings Adjustment

Adjust systemd configuration on K3s nodes to bind to the new VLAN 20 IPs.

1. Master Node NUC configuration (/etc/systemd/system/k3s.service):

# Inside ExecStart command, append node-ip and advertise-address variables ExecStart=/usr/local/bin/k3s server \ --node-ip=192.168.20.21 \ --advertise-address=192.168.20.21 \ --flannel-backend=wireguard-native

2. Worker Node configuration (/etc/systemd/system/k3s-agent.service.env on e.g., kubernetes1):

K3S_URL="https://192.168.20.21:6443" K3S_TOKEN="K101..." K3S_AGENT_ARGS="--node-ip=192.168.20.22"

3. Reload & Restart services:

sudo systemctl daemon-reload sudo systemctl restart k3s-agent # or k3s for master
7
Phase 7: Validation & Diagnostics

Execute routing verification trace from your workstation (HP-2):

tracert 192.168.20.22

Verify paths are correctly localized (Hop 1: Switch SVI 192.168.30.99, Hop 2: worker node 192.168.20.22).

Execute health checks on the cluster:

pwsh -File .\Test-K3sClusterHealth.ps1
