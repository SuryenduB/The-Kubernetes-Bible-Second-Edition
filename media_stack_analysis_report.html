☸️ Media Stack - Kubernetes Security & Best Practices Analysis Report

Date: June 09, 2026 | Auditor: Antigravity (using kubernetes-specialist skill) | Target Directory: kubernetes-manifests/media/

📋 Executive Summary

An audit of the media stack manifests (audiobookshelf and calibre-web) was conducted using the Kubernetes Specialist guidelines.

Compared to the base iiqstack manifests, the media stack is significantly better configured:

  • Dedicated Namespace: Deploys in an isolated media namespace.
  • Probes and Health Checks: Fully defines liveness and readiness probes.
  • Resource Controls: Sets resource requests and limits on core containers.
  • Deterministic Rollouts: Uses specific versions instead of latest tags (e.g. 2.35.1 and 0.6.25-ls358).

However, two critical anti-patterns were discovered within the initialization logic of calibre-web.yaml that can block pod startups and violate isolation.

🔍 Detailed Audits

1. Calibre-Web Workload (calibre-web.yaml)

⚠️
WARNING: While Calibre-Web is properly resource-constrained and monitored, the custom init container poses a major reliability bottleneck.
  • Startup Network Blockers:
    apk add --no-cache wget sqlite
wget https://github.com/janeczku/calibre-web/raw/master/library/metadata.db ...
    If the cluster runs in an air-gapped environment, experiences a temporary DNS outage, or GitHub/Alpine repositories are down, the init container will crash, causing a perpetual Init:CrashLoopBackOff loop.
  • NFS I/O Bottlenecks:
    chown -R 1000:1000 /library
    Running recursive ownership changes on a network-shared file system (NFS) with hundreds of gigabytes of files results in massive I/O load and extremely slow startup times, which can exceed the Kubelet container startup limits.

2. Audiobookshelf Workload (audiobookshelf.yaml)

💡
TIP: This manifest is a textbook example of a well-configured container, featuring healthy resource limits, probes, and an integrated PodDisruptionBudget.
  • Excellent Features:
    • Resource Boundaries: Requests 100m/256Mi and limits 1/1Gi.
    • Probes: Configures robust liveness and readiness endpoints on /healthcheck.
    • Resilience: Configures a PodDisruptionBudget ensuring high availability during node drainage.

🛠️ Refactoring Recommendations

To align the media stack with the Kubernetes Specialist guidelines:

A. Pre-package the Init Container Image

Avoid running package managers at startup. Build a custom mini-image containing sqlite and wget beforehand, and push it to your local registry (e.g. 192.168.0.236:5000/calibre-init:1.0):

FROM alpine:3.21
RUN apk add --no-cache sqlite wget

Then use this deterministic image in calibre-web.yaml:

initContainers:
  - name: init-library
    image: 192.168.0.236:5000/calibre-init:1.0
    command: ["sh", "-c"]
    # Removed apk add wget sqlite

B. Eliminate Expensive chown

Rather than running chown -R on network storage, use K3s/Kubernetes native security context to map volume ownership at the pod level:

securityContext:
  runAsUser: 1000
  runAsGroup: 1000
  fsGroup: 1000

This forces the container runtime to handle permissions on mounting without recursive traversal over network volumes.
