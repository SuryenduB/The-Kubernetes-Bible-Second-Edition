# Code Changes for OpenLingo Translation & Console Hang Issues

The following backend fixes have been applied:

1. **`lib/ai/models.ts`**
   - Updated line 59: Changed `nvidiaProvider(modelInfo.internalId)` to `nvidiaProvider.chat(modelInfo.internalId)` to use the AI SDK chat model method properly.

2. **`lib/nvidia-fix.ts`**
   - Modified the global interceptor to dynamically manage request payloads:
     - For `"minimaxai/minimax-m3"`, stripped `"tools"`, `"tool_choice"`, and `"response_format"` parameters to prevent bad request errors (as it lacks native support).
     - For `"deepseek-ai/deepseek-v4-pro"`, kept tools enabling tool calling capability (e.g. `readArticle`).
     - Added case-insensitive cleaning of any existing `"authorization"` headers (in whatever capitalization they arrive, e.g., `Authorization`, `authorization`) before forcing/setting the lowercase `"authorization"` header.

3. **`app/api/chat/route.ts`**
   - Reverted the direct fetch proxy route back to Vercel AI SDK's standard `streamText` function.
   - Registered the tools generated by `createTools` dynamically.
   - Streamed the response using `result.toDataStreamResponse()` to return it in the Vercel AI SDK Data Stream protocol format expected by `useChat` on the frontend, resolving the console hang/parsing issue.

4. **`lib/article/translate.ts`**
   - Updated the Gemini translation model at line 113 from `"gemini-3-flash-preview"` to `"gemini-2.5-flash"`.
