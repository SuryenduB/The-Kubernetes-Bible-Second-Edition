🔒 Security Posture Note

VLAN 20 (K3s Production) is completely isolated at Layer 3. The switch will block any outbound connections from VLAN 20 to your Management VLAN 10 to ensure a compromised container cannot reach key systems.

⚙️ Switch L3 Configuration Steps

Configure the following in the Netgear Switch Web Management GUI:

  1. Enable Global IP Routing:

    Navigate to Routing > IP > IP Configuration, set Routing Mode to Enable, and click apply.

  2. Create Switched Virtual Interfaces (SVIs):

    Go to Routing > IP > IP Interface Configuration, and create routing interface addresses for each VLAN:

    • VLAN 1 Interface: 192.168.0.99 (Subnet Mask: 255.255.255.0)
    • VLAN 10 Interface: 192.168.10.99 (Subnet Mask: 255.255.255.0)
    • VLAN 20 Interface: 192.168.20.99 (Subnet Mask: 255.255.255.0)
    • VLAN 30 Interface: 192.168.30.99 (Subnet Mask: 255.255.255.0)
  3. Set Default Route for Internet Egress:

    Navigate to Routing > IP > Static Route Configuration and add the NUC as the default gateway:

    Route Type: Default Destination IP: 0.0.0.0 Subnet Mask: 0.0.0.0 Next Hop Gateway: 192.168.0.21 (NUC VLAN 1 Interface - acting as NAT Gateway)
  4. DHCP Relay Settings:

    Enable DHCP relay helper under System > Services > DHCP Relay and point the helper address to 192.168.0.1 so your router can serve DHCP leases dynamically for VLAN 20 and 30.

🌐 NAT Gateway Routing Tables (NUC Config)

Since the Vodafone Router (192.168.0.1) is locked down, return routing for VLAN 10 and 30 is handled by static routes on the NUC pointing to the switch (192.168.0.99). Configure these routes in your NUC netplan configuration:

Destination Network Subnet Mask Next Hop Gateway (Switch IP) Description
192.168.10.0 255.255.255.0 192.168.0.99 Routes return traffic back to MGMT network
192.168.20.0 255.255.255.0 192.168.0.99 Routes return traffic back to K3s Production network
192.168.30.0 255.255.255.0 192.168.0.99 Routes return traffic back to Trusted Devices

💡 Optimization Benefit

Because local inter-VLAN routing is handled inside the switch, a node on 192.168.20.19 sending database files to 192.168.20.128 routes at 1 Gbps wire-speed without ever leaving the switch backbone!
